# GET /relationships/invitationsSummaryV2

Obtiene un resumen del número total de invitaciones de conexión enviadas que están pendientes de aceptación.

## Endpoint

```
GET https://www.linkedin.com/voyager/api/relationships/invitationsSummaryV2
```

## Query Parameters

| Parámetro | Tipo | Requerido | Descripción |
|-----------|------|-----------|-------------|
| `types` | string | Sí | Lista de tipos de invitaciones. Usar `List(SENT_INVITATION_COUNT)` |

## Headers

```
Cookie: li_at={token}; JSESSIONID=ajax:{session}
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
X-Li-User-Agent: LIAuthLibrary:0.0.3 com.linkedin.android:4.1.881
csrf-token: {session}
```

## Ejemplo de Request

<CodeGroup>

```bash cURL
curl -X GET "https://www.linkedin.com/voyager/api/relationships/invitationsSummaryV2?types=List(SENT_INVITATION_COUNT)" \\
  -H "Cookie: li_at=YOUR_LI_AT; JSESSIONID=ajax:YOUR_JSESSIONID" \\
  -H "User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36" \\
  -H "X-Li-User-Agent: LIAuthLibrary:0.0.3 com.linkedin.android:4.1.881" \\
  -H "csrf-token: YOUR_JSESSIONID"
```

```python Python
import requests

li_at = "YOUR_LI_AT"
jsessionid = "YOUR_JSESSIONID"

headers = {
    'Cookie': f'li_at={li_at}; JSESSIONID=ajax:{jsessionid}',
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36',
    'X-Li-User-Agent': 'LIAuthLibrary:0.0.3 com.linkedin.android:4.1.881',
    'csrf-token': jsessionid
}

response = requests.get(
    'https://www.linkedin.com/voyager/api/relationships/invitationsSummaryV2',
    headers=headers,
    params={'types': 'List(SENT_INVITATION_COUNT)'}
)

data = response.json()
print(f"Pending invitations: {data['numTotalSentInvitations']}")
```

```javascript JavaScript
const li_at = 'YOUR_LI_AT';
const jsessionid = 'YOUR_JSESSIONID';

const headers = {
  'Cookie': `li_at=${li_at}; JSESSIONID=ajax:${jsessionid}`,

```

## Respuesta Exitosa

```json
{
  "numTotalSentInvitations": 47,
  "numSentConnectionInvitations": 47,
  "$type": "com.linkedin.voyager.relationships.InvitationsSummary"
}
```

### Campos de Respuesta

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `numTotalSentInvitations` | integer | Número total de invitaciones enviadas pendientes |
| `numSentConnectionInvitations` | integer | Número de invitaciones de conexión pendientes |
| `$type` | string | Tipo del objeto de respuesta |

## Casos de Uso

Este endpoint es útil para:

- Verificar cuántas invitaciones tienes pendientes antes de enviar más
- Monitorear tu cuota de invitaciones (LinkedIn limita a ~100-200 pendientes)
- Implementar lógica de rate limiting en tu aplicación
- Decidir si pausar el envío de invitaciones

<Callout kind="info">
  LinkedIn limita el númerode invitaciones pendientes que puedes tener simultáneamente (típicamente entre 100-200). Este endpoint te ayuda a monitorear ese límite.
</Callout>

## Rate Limits

- **Frecuencia recomendada**: No más de 1 request por minuto
- **Throttling**: LinkedIn puede bloquear si se consulta muy frecuentemente

<Callout kind="tip">
  Este endpoint es más ligero que `/relationships/sentInvitationViewsV2` cuando solo necesitas el conteo total.
</Callout>

## Errores Comunes

| Código | Descripción | Solución |
|--------|-------------|----------|
| 401 | No autorizado | Verificar cookies li_at y JSESSIONID |
| 403 | Prohibido | Revisar headers CSRF y User-Agent |
| 429 | Rate limit excedido | Esperar antes de reintentar |
